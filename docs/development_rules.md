# 開発ルール

## 基本方針

* GitHubでソースコードを管理する
* 作業前に最新の状態を取得する
* 大きな変更を行う場合は事前に相談する
* 不明な点は勝手に実装せず相談する
* 実行前にpip install -r requirements.txtを行う．
* 何かpip installしたら，報告する．

## ファイル管理

* .env はGitHubへアップロードしない
* .DS_Store や .Rhistory はアップロードしない

## ドキュメント管理

* 要件変更があった場合は requirements.md を更新する
* 機能追加があった場合は incremental_plan.md を更新する
